# Website architecture

The **YAT scenario website** (`diploma-cloud-cyber-website`, a sibling repo) is the in-world delivery vehicle and the **single source of truth for scenario content** — the clusters pull from it, and the old `<repo_root>/scenario/` folder is no longer authoritative. It is an Astro static site deployed to Cloudflare Pages, comprising a public marketing site plus an `/intranet/` that the assessments reference. This document describes its durable architecture: the tech stack, the state-folder URL model, content collections and the shared schema, the projects model, state-versioned docs, the year-offset pattern, downloadable documents, the public site and sign-in, the diagram pattern, Astro gotchas, and the standing principles applied.

Paths are relative to the `diploma-cloud-cyber-website/` repo root.

## Tech stack + single-source configs
- **Astro** static site, **TypeScript strict**; deploys to Cloudflare Pages.
- `src/content.config.ts` (Astro v6 location — not `src/content/config.ts`) — collections via `glob()`.
- `src/config/states.ts` — single source of truth for cluster/AT **states**: `STATES[]` (real,
  navigable) + `PLACEHOLDER_CLUSTERS[]` (future-cluster slots in the picker).
- `src/config/projects.ts` — single source of truth for **projects** (`PROJECTS[]` + per-state
  `stateOverrides` via `getProjectStatus`/`getProjectTimeline`, so a project can read Current in one
  state and Completed in a later one).
- `@astrojs/sitemap` + `public/robots.txt` exclude `/intranet/` and `/sign-in`.

## State-folder URL model (intranet)
Every intranet URL is state-prefixed, e.g. `/intranet/s1-cl1-at1/policies/whs`. **One markdown doc
renders at multiple URLs** — one per state in its `appearsIn` list — so the user stays "in their state"
while navigating. Routes: `/intranet/` (state picker: Semester→Cluster→AT) → `/intranet/[state]/` →
`/intranet/[state]/{category}/` → `/intranet/[state]/{category}/[item]`; projects at
`/intranet/[state]/projects/[project]/[item]`. State shows as an italic line under each `<h1>`.

## Content collections + schema
Collections (glob loader): `policies`, `reference`, `projects` (path-encoded `project-slug/doc-slug`),
`ict` (YAT-internal operational docs). **Shared `docSchema`:** `title`, `description`, `appearsIn`
(state slugs), optional `uocReferences`, `placeholder`, `order` (sort), `yearOffset`.

## Projects model
Projects attach to a state by `appearsIn`. The standing projects + their scenario roles:
- **MTS – LMS Cloud Infrastructure** (`lms-cloud-infrastructure`) — the **assessment** engagement (CL1).
- **AccentLoitte – LMS Replacement** (`lms-replacement`) — an **exemplar past project** (2022).
- **MTS – Accounting System Cloud Migration** (`accounting-cloud-migration`) — the **practice**
  engagement (Ledgerline), a peer engagement to the LMS (see the S1 CL1 delivery process in
  [process-delivery.md](process-delivery.md)).
- **LMS Global Expansion** (`lms-global-expansion`) — the CL2 web-scale + DR vehicle (see the S1 CL2
  assessment flow in [scenario-flow.md](scenario-flow.md)).

## State-versioned docs
Some ICT docs share a title but **fork content by state** — on-prem (AT1/AT2) / post-cutover single-AZ
(AT3) / HA-hardened Multi-AZ (CL2-AT1, CL3-AT1): Network Diagram, ICT Environment Overview,
Hardware/Software Inventory, LMS Server→Infrastructure Specs, plus the hosting-forked LMS Application
Spec (`lms-application-spec.md` on-prem / `lms-application-spec-cloud.md` cloud). **One-DR-Plan model:**
a single DR-Plan document/template across clusters (current vs deprecated banner by state).

## Year-offset evergreen pattern
Year-graded docs use `yearOffset: 0/-1/-2`; `[item].astro` computes `currentYear + yearOffset` and
prepends it to the title. Files stay stable; year labels rotate with the calendar.

## Downloadable documents
Branded artefacts served from `public/` (generated by the source repo's `scripts/` — see
[document-template-system.md](document-template-system.md)): `public/templates/` = fillable templates
(Business Case, BC Presentation, Solution Design, Deployment Report, DR Plan), on the intranet Templates
page; `public/documents/` = student-model PDFs (+ editable `.docx`), e.g. the AccentLoitte exemplar
chain + supplied baseline designs.

## Public site + sign-in
Public marketing site (homepage, study areas, about, apply, contact, locations, sign-in, intranet stub,
404) on brand pack v1 (Source Sans 3), persistent disclosure banner. Mock SSO at `/sign-in` (GET to
`/intranet/`, no real auth); the user then picks their state.

## Diagram pattern
Diagrams authored in **draw.io**: export **SVG** for page embed + keep the **`.drawio` XML** for
download (network diagrams in `public/diagrams/`: on-prem / post-cutover / HA-hardened).

## Astro patterns / gotchas
- **`getStaticPaths` runs in isolated scope** — module-level `const`s are NOT visible inside; top-level
  `import`s ARE.
- **No `require()`** — ES modules only (Astro/Vite).
- **Content API (v6):** `getCollection`, `render` from `astro:content`; entries have `.id` (not
  `.slug`); `const { Content } = await render(entry)`.
- **Stale dev-server CSS** after big scoped-style changes — hard-refresh or restart `npm run dev`.

## Standing principles applied
- **In-world only on intranet** — no course/assessment/cluster meta-language (sole exception: the small
  UoC footer on migrated docs).
- **Single source of truth per document** — one markdown, many state URLs.
- **Draft naming pragmatism** — draft artefacts keep their natural, in-world names rather than being
  renamed to signal draft status; the draft state lives in the workflow, not the filename.
