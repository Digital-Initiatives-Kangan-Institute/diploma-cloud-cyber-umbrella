The YAT scenario website is the in-world delivery vehicle for the cluster scenario. **Live at `diploma-cloud-cyber-website/` (sibling repo to `diploma-cloud-cyber`).** Astro static site, deploys to Cloudflare Pages. Public site + intranet under `/intranet/`, with the intranet structured as state-prefixed folders per cluster-AT.

## Tech stack + key files

- **Astro 6.3.7**, TypeScript strict
- `src/content.config.ts` (NOT `src/content/config.ts` — Astro v6 location) — content-collection definitions using the `glob()` loader
- `src/config/states.ts` — single source of truth for cluster/AT states; defines `STATES[]` (real, navigable) + `PLACEHOLDER_CLUSTERS[]` (future-cluster slots shown in state picker)
- `src/config/projects.ts` — single source of truth for projects; `PROJECTS[]` with metadata (slug, displayName, consultancy, status, timeline, summary, appearsIn)
- `astro.config.mjs` — `@astrojs/sitemap` integration with intranet + sign-in excluded from sitemap
- `public/robots.txt` — Disallows `/intranet/` and `/sign-in`

## State-folder URL model (intranet)

Every intranet URL is prefixed with state, e.g. `/intranet/s1-cl1-at1/policies/whs`. Single source of truth per document — same markdown content renders at multiple URLs (one per state in its `appearsIn` list). User stays "in their state" while navigating.

- `/intranet/` — state picker landing (hierarchical: Semester → Cluster → AT). Real ATs clickable; future clusters/ATs as italic placeholder rows.
- `/intranet/[state]/` — per-state intranet landing (5-category bullet list)
- `/intranet/[state]/{category}/` — per-state category landing (lists items in collection for that state)
- `/intranet/[state]/{category}/[item]` — per-state document page
- Project sub-paths: `/intranet/[state]/projects/[project]/` and `/intranet/[state]/projects/[project]/[item]`

State context appears as italic `state-notation` paragraph under each `<h1>`.

## Year-offset evergreen pattern

Used for cost-baseline-style docs that have year-graded content. Three files (`lms-costing.md`, `lms-costing-prior.md`, `lms-costing-prior-2.md`) with `yearOffset: 0 / -1 / -2`. The dynamic route at `[item].astro` computes `new Date().getFullYear() + yearOffset` and prepends to the rendered title. Files stay stable across years; year labels rotate with the calendar.

## Astro patterns / gotchas learned

- **`getStaticPaths` runs in isolated scope** — module-level `const` declarations are NOT visible inside; top-level `import` statements ARE. (Burnt on [area].astro initially.)
- **No `require()`** in Astro/Vite — ES modules only.
- **Content-collection API in v6**: import `getCollection`, `render` from `'astro:content'`; entries have `.id` (not `.slug`); render via `const { Content } = await render(entry)`.
- **Stale dev-server CSS** after big scoped-style changes: hard-refresh OR restart `npm run dev`. First diagnostic for "this CSS clearly isn't matching what I wrote".
- **`z` from `astro:content`** is deprecated in Astro v6 (use a different import); non-blocking hint, future cleanup.
